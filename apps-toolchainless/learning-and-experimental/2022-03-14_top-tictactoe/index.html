This web app follows Project: Tic Tac Toe from The Odin Project, including the AI extension criteria.

This app wasn't designed with mobile in mind.

Five different AI players are implemented:

  • Hard: Takes advantage of guaranteed wins, or forces a draw if there isn't one. Cannot lose.
    • Implemented using the minimax algorithm.
    • This AI may sometimes choose not to take a win on the next turn, but (I believe) it will only do so if it can guarantee a win in subsequent turns anyway.
  • Medium (Shallow): An intelligent but flawed AI whose weakness is that it only looks one step ahead.
    • The AI first checks if it can win on the current turn. If so, it takes the win.
    • The AI then checks if its opponent can win on the current turn. If so, it blocks at least one of its opponent's possible winning moves, meaning if there are multiple possible winning moves, its opponent can still win.
    • The AI then checks if there's any possible path to victory. If so, then it will try to "win as fast as possible", favouring lines that already contain its own marker.
    • As a last resort, the AI picks any random empty square.
  • Medium (Random): A sabotaged version of Hard that has a chance of playing poorly on each turn.
    • During each turn, there's a 70% chance of playing like Hard, and a 30% chance of playing like Easy.
  • Easy: Picks any empty square at random. All empty squares have equal probabilities of being chosen.
  • Loser: Tries to lose. Also implemented using the minimax algorithm, so it behaves like Hard, but for a different goal.

Although the Hard and Loser AIs' minimax algorithm is very good, there's still room for improvement. Currently, if the algorithm cannot guarantee a win, it will randomly pick between moves that can guarantee a draw, rather than trying to pick the "best" move between them. (I might look into better algorithms to do this!)
